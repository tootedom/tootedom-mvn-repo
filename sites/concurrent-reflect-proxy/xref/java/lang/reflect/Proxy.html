52  /**
53   * {@code Proxy} provides static methods for creating dynamic proxy
54   * classes and instances, and it is also the superclass of all
55   * dynamic proxy classes created by those methods.
56   *
57   * <p>To create a proxy for some interface {@code Foo}:
58   * <pre>
59   *     InvocationHandler handler = new MyInvocationHandler(...);
60   *     Class proxyClass = Proxy.getProxyClass(
61   *         Foo.class.getClassLoader(), new Class[] { Foo.class });
62   *     Foo f = (Foo) proxyClass.
63   *         getConstructor(new Class[] { InvocationHandler.class }).
64   *         newInstance(new Object[] { handler });
65   * </pre>
66   * or more simply:
67   * <pre>
68   *     Foo f = (Foo) Proxy.newProxyInstance(Foo.class.getClassLoader(),
69   *                                          new Class[] { Foo.class },
70   *                                          handler);
71   * </pre>
72   *
73   * <p>A <i>dynamic proxy class</i> (simply referred to as a <i>proxy
74   * class</i> below) is a class that implements a list of interfaces
75   * specified at runtime when the class is created, with behavior as
76   * described below.
77   *
78   * A <i>proxy interface</i> is such an interface that is implemented
79   * by a proxy class.
80   *
81   * A <i>proxy instance</i> is an instance of a proxy class.
82   *
83   * Each proxy instance has an associated <i>invocation handler</i>
84   * object, which implements the interface {@link InvocationHandler}.
85   * A method invocation on a proxy instance through one of its proxy
86   * interfaces will be dispatched to the {@link InvocationHandler#invoke
87   * invoke} method of the instance's invocation handler, passing the proxy
88   * instance, a {@code java.lang.reflect.Method} object identifying
89   * the method that was invoked, and an array of type {@code Object}
90   * containing the arguments.  The invocation handler processes the
91   * encoded method invocation as appropriate and the result that it
92   * returns will be returned as the result of the method invocation on
93   * the proxy instance.
94   *
95   * <p>A proxy class has the following properties:
96   *
97   * <ul>
98   * <li>Proxy classes are public, final, and not abstract.
99   *
100  * <li>The unqualified name of a proxy class is unspecified.  The space
101  * of class names that begin with the string {@code "$Proxy"}
102  * should be, however, reserved for proxy classes.
103  *
104  * <li>A proxy class extends {@code java.lang.reflect.Proxy}.
105  *
106  * <li>A proxy class implements exactly the interfaces specified at its
107  * creation, in the same order.
108  *
109  * <li>If a proxy class implements a non-public interface, then it will
110  * be defined in the same package as that interface.  Otherwise, the
111  * package of a proxy class is also unspecified.  Note that package
112  * sealing will not prevent a proxy class from being successfully defined
113  * in a particular package at runtime, and neither will classes already
114  * defined by the same class loader and the same package with particular
115  * signers.
116  *
117  * <li>Since a proxy class implements all of the interfaces specified at
118  * its creation, invoking {@code getInterfaces} on its
119  * {@code Class} object will return an array containing the same
120  * list of interfaces (in the order specified at its creation), invoking
121  * {@code getMethods} on its {@code Class} object will return
122  * an array of {@code Method} objects that include all of the
123  * methods in those interfaces, and invoking {@code getMethod} will
124  * find methods in the proxy interfaces as would be expected.
125  *
126  * <li>The {@link Proxy#isProxyClass Proxy.isProxyClass} method will
127  * return true if it is passed a proxy class-- a class returned by
128  * {@code Proxy.getProxyClass} or the class of an object returned by
129  * {@code Proxy.newProxyInstance}-- and false otherwise.
130  *
131  * <li>The {@code java.security.ProtectionDomain} of a proxy class
132  * is the same as that of system classes loaded by the bootstrap class
133  * loader, such as {@code java.lang.Object}, because the code for a
134  * proxy class is generated by trusted system code.  This protection
135  * domain will typically be granted
136  * {@code java.security.AllPermission}.
137  *
138  * <li>Each proxy class has one public constructor that takes one argument,
139  * an implementation of the interface {@link InvocationHandler}, to set
140  * the invocation handler for a proxy instance.  Rather than having to use
141  * the reflection API to access the public constructor, a proxy instance
142  * can be also be created by calling the {@link Proxy#newProxyInstance
143  * Proxy.newProxyInstance} method, which combines the actions of calling
144  * {@link Proxy#getProxyClass Proxy.getProxyClass} with invoking the
145  * constructor with an invocation handler.
146  * </ul>
147  *
148  * <p>A proxy instance has the following properties:
149  *
150  * <ul>
151  * <li>Given a proxy instance {@code proxy} and one of the
152  * interfaces implemented by its proxy class {@code Foo}, the
153  * following expression will return true:
154  * <pre>
155  *     {@code proxy instanceof Foo}
156  * </pre>
157  * and the following cast operation will succeed (rather than throwing
158  * a {@code ClassCastException}):
159  * <pre>
160  *     {@code (Foo) proxy}
161  * </pre>
162  *
163  * <li>Each proxy instance has an associated invocation handler, the one
164  * that was passed to its constructor.  The static
165  * {@link Proxy#getInvocationHandler Proxy.getInvocationHandler} method
166  * will return the invocation handler associated with the proxy instance
167  * passed as its argument.
168  *
169  * <li>An interface method invocation on a proxy instance will be
170  * encoded and dispatched to the invocation handler's {@link
171  * InvocationHandler#invoke invoke} method as described in the
172  * documentation for that method.
173  *
174  * <li>An invocation of the {@code hashCode},
175  * {@code equals}, or {@code toString} methods declared in
176  * {@code java.lang.Object} on a proxy instance will be encoded and
177  * dispatched to the invocation handler's {@code invoke} method in
178  * the same manner as interface method invocations are encoded and
179  * dispatched, as described above.  The declaring class of the
180  * {@code Method} object passed to {@code invoke} will be
181  * {@code java.lang.Object}.  Other public methods of a proxy
182  * instance inherited from {@code java.lang.Object} are not
183  * overridden by a proxy class, so invocations of those methods behave
184  * like they do for instances of {@code java.lang.Object}.
185  * </ul>
186  *
187  * <h3>Methods Duplicated in Multiple Proxy Interfaces</h3>
188  *
189  * <p>When two or more interfaces of a proxy class contain a method with
190  * the same name and parameter signature, the order of the proxy class's
191  * interfaces becomes significant.  When such a <i>duplicate method</i>
192  * is invoked on a proxy instance, the {@code Method} object passed
193  * to the invocation handler will not necessarily be the one whose
194  * declaring class is assignable from the reference type of the interface
195  * that the proxy's method was invoked through.  This limitation exists
196  * because the corresponding method implementation in the generated proxy
197  * class cannot determine which interface it was invoked through.
198  * Therefore, when a duplicate method is invoked on a proxy instance,
199  * the {@code Method} object for the method in the foremost interface
200  * that contains the method (either directly or inherited through a
201  * superinterface) in the proxy class's list of interfaces is passed to
202  * the invocation handler's {@code invoke} method, regardless of the
203  * reference type through which the method invocation occurred.
204  *
205  * <p>If a proxy interface contains a method with the same name and
206  * parameter signature as the {@code hashCode}, {@code equals},
207  * or {@code toString} methods of {@code java.lang.Object},
208  * when such a method is invoked on a proxy instance, the
209  * {@code Method} object passed to the invocation handler will have
210  * {@code java.lang.Object} as its declaring class.  In other words,
211  * the public, non-final methods of {@code java.lang.Object}
212  * logically precede all of the proxy interfaces for the determination of
213  * which {@code Method} object to pass to the invocation handler.
214  *
215  * <p>Note also that when a duplicate method is dispatched to an
216  * invocation handler, the {@code invoke} method may only throw
217  * checked exception types that are assignable to one of the exception
218  * types in the {@code throws} clause of the method in <i>all</i> of
219  * the proxy interfaces that it can be invoked through.  If the
220  * {@code invoke} method throws a checked exception that is not
221  * assignable to any of the exception types declared by the method in one
222  * of the proxy interfaces that it can be invoked through, then an
223  * unchecked {@code UndeclaredThrowableException} will be thrown by
224  * the invocation on the proxy instance.  This restriction means that not
225  * all of the exception types returned by invoking
226  * {@code getExceptionTypes} on the {@code Method} object
227  * passed to the {@code invoke} method can necessarily be thrown
228  * successfully by the {@code invoke} method.
229  *
230  * @author      Peter Jones
231  * @see         InvocationHandler
232  * @since       1.3
233  */
234 public class Proxy implements java.io.Serializable {

308     /**
309      * Returns the {@code java.lang.Class} object for a proxy class
310      * given a class loader and an array of interfaces.  The proxy class
311      * will be defined by the specified class loader and will implement
312      * all of the supplied interfaces.  If a proxy class for the same
313      * permutation of interfaces has already been defined by the class
314      * loader, then the existing proxy class will be returned; otherwise,
315      * a proxy class for those interfaces will be generated dynamically
316      * and defined by the class loader.
317      *
318      * <p>There are several restrictions on the parameters that may be
319      * passed to {@code Proxy.getProxyClass}:
320      *
321      * <ul>
322      * <li>All of the {@code Class} objects in the
323      * {@code interfaces} array must represent interfaces, not
324      * classes or primitive types.
325      *
326      * <li>No two elements in the {@code interfaces} array may
327      * refer to identical {@code Class} objects.
328      *
329      * <li>All of the interface types must be visible by name through the
330      * specified class loader.  In other words, for class loader
331      * {@code cl} and every interface {@code i}, the following
332      * expression must be true:
333      * <pre>
334      *     Class.forName(i.getName(), false, cl) == i
335      * </pre>
336      *
337      * <li>All non-public interfaces must be in the same package;
338      * otherwise, it would not be possible for the proxy class to
339      * implement all of the interfaces, regardless of what package it is
340      * defined in.
341      *
342      * <li>For any set of member methods of the specified interfaces
343      * that have the same signature:
344      * <ul>
345      * <li>If the return type of any of the methods is a primitive
346      * type or void, then all of the methods must have that same
347      * return type.
348      * <li>Otherwise, one of the methods must have a return type that
349      * is assignable to all of the return types of the rest of the
350      * methods.
351      * </ul>
352      *
353      * <li>The resulting proxy class must not exceed any limits imposed
354      * on classes by the virtual machine.  For example, the VM may limit
355      * the number of interfaces that a class may implement to 65535; in
356      * that case, the size of the {@code interfaces} array must not
357      * exceed 65535.
358      * </ul>
359      *
360      * <p>If any of these restrictions are violated,
361      * {@code Proxy.getProxyClass} will throw an
362      * {@code IllegalArgumentException}.  If the {@code interfaces}
363      * array argument or any of its elements are {@code null}, a
364      * {@code NullPointerException} will be thrown.
365      *
366      * <p>Note that the order of the specified proxy interfaces is
367      * significant: two requests for a proxy class with the same combination
368      * of interfaces but in a different order will result in two distinct
369      * proxy classes.
370      *
371      * @param   loader the class loader to define the proxy class
372      * @param   interfaces the list of interfaces for the proxy class
373      *          to implement
374      * @return  a proxy class that is defined in the specified class loader
375      *          and that implements the specified interfaces
376      * @throws  IllegalArgumentException if any of the restrictions on the
377      *          parameters that may be passed to {@code getProxyClass}
378      *          are violated
379      * @throws  NullPointerException if the {@code interfaces} array
380      *          argument or any of its elements are {@code null}
381      */
382     public static Class<?> getProxyClass(ClassLoader loader,
383                                          Class<?>... interfaces)
384             throws IllegalArgumentException
385     {
386 
387         if (interfaces.length > 65535) {
388             throw new IllegalArgumentException("interface limit exceeded");
389         }
390 
391         Class<?> proxyClass = null;
392 
393         /* collect interface names to use as key for proxy class cache */
394         String[] interfaceNames = new String[interfaces.length];
395 
396         // for detecting duplicates
397         Set<Class<?>> interfaceSet = new HashSet();
398 
399         for (int i = 0; i < interfaces.length; i++) {
400             /*
401              * Verify that the class loader resolves the name of this
402              * interface to the same Class object.
403              */
404             String interfaceName = interfaces[i].getName();
405             Class<?> interfaceClass = null;
406             try {
407                 interfaceClass = Class.forName(interfaceName, false, loader);
408             } catch (ClassNotFoundException e) {
409             }
410             if (interfaceClass != interfaces[i]) {
411                 throw new IllegalArgumentException(
412                         interfaces[i] + " is not visible from class loader");
413             }
414 
415             /*
416              * Verify that the Class object actually represents an
417              * interface.
418              */
419             if (!interfaceClass.isInterface()) {
420                 throw new IllegalArgumentException(
421                         interfaceClass.getName() + " is not an interface");
422             }
423 
424             /*
425              * Verify that this interface is not a duplicate.
426              */
427             if (interfaceSet.contains(interfaceClass)) {
428                 throw new IllegalArgumentException(
429                         "repeated interface: " + interfaceClass.getName());
430             }
431             interfaceSet.add(interfaceClass);
432 
433             interfaceNames[i] = interfaceName;
434         }
435 
436         /*
437          * Using string representations of the proxy interfaces as
438          * keys in the proxy class cache (instead of their Class
439          * objects) is sufficient because we require the proxy
440          * interfaces to be resolvable by name through the supplied
441          * class loader, and it has the advantage that using a string
442          * representation of a class makes for an implicit weak
443          * reference to the class.
444          */
445         List<String> key = Arrays.asList(interfaceNames);
446 
447         /*
448          * Find or create the proxy class cache for the class loader.
449          */
450         ConcurrentHashMap<List<String>, Object> cache;
451         Object loaderKey = loader==null ? NULL_KEY: loader;
452         cache = loaderToCache.get(loaderKey);
453         if (cache == null) {
454             cache = new ConcurrentHashMap<List<String>, Object>();
455 
456             loaderToCache.put(loaderKey, cache);
457         }
458             /*
459              * This mapping will remain valid for the duration of this
460              * method, without further synchronization, because the mapping
461              * will only be removed if the class loader becomes unreachable.
462              */
463 
464 
465         /*
466          * Look up the list of interfaces in the proxy class cache using
467          * the key.  This lookup will result in one of three possible
468          * kinds of values:
469          *     null, if there is currently no proxy class for the list of
470          *         interfaces in the class loader,
471          *     the pendingGenerationMarker object, if a proxy class for the
472          *         list of interfaces is currently being generated,
473          *     or a weak reference to a Class object, if a proxy class for
474          *         the list of interfaces has already been generated.
475          */
476             /*
477              * Note that we need not worry about reaping the cache for
478              * entries with cleared weak references because if a proxy class
479              * has been garbage collected, its class loader will have been
480              * garbage collected as well, so the entire cache will be reaped
481              * from the loaderToCache map.
482              */
483         do {
484             Object value = cache.get(key);
485             if (value instanceof Reference) {
486                 proxyClass = (Class<?>) ((Reference) value).get();
487             }
488 
489             if (proxyClass != null) {
490                 // proxy class already generated: return it
491                 return proxyClass;
492             } else if (value == pendingGenerationMarker) {
493                 // proxy class being generated: wait for it
494                 // spin and check if created
495                 spinningWaitStrategy.dowait(cache,key,pendingGenerationMarker);
496                 continue;
497             } else {
498                     /*
499                      * No proxy class for this list of interfaces has been
500                      * generated or is being generated, so we will go and
501                      * generate it now.  Mark it as pending generation.
502                      */
503                 value = cache.putIfAbsent(key, pendingGenerationMarker);
504                 if(value==null) {
505                     break;
506                 }
507             }
508         } while (true);
509 
510 
511         try {
512             String proxyPkg = null;     // package to define proxy class in
513 
514             /*
515              * Record the package of a non-public proxy interface so that the
516              * proxy class will be defined in the same package.  Verify that
517              * all non-public proxy interfaces are in the same package.
518              */
519             for (int i = 0; i < interfaces.length; i++) {
520                 int flags = interfaces[i].getModifiers();
521                 if (!Modifier.isPublic(flags)) {
522                     String name = interfaces[i].getName();
523                     int n = name.lastIndexOf('.');
524                     String pkg = ((n == -1) ? "" : name.substring(0, n + 1));
525                     if (proxyPkg == null) {
526                         proxyPkg = pkg;
527                     } else if (!pkg.equals(proxyPkg)) {
528                         throw new IllegalArgumentException(
529                                 "non-public interfaces from different packages");
530                     }
531                 }
532             }
533 
534             if (proxyPkg == null) {     // if no non-public proxy interfaces,
535                 proxyPkg = "";          // use the unnamed package
536             }
537 
538             {
539                 /*
540                  * Choose a name for the proxy class to generate.
541                  */
542 
543                 long num = nextUniqueNumber.getAndIncrement();
544 
545                 String proxyName = proxyPkg + proxyClassNamePrefix + num;
546                 /*
547                  * Verify that the class loader hasn't already
548                  * defined a class with the chosen name.
549                  */
550 
551                 /*
552                  * Generate the specified proxy class.
553                  */
554                 byte[] proxyClassFile = ProxyGenerator.generateProxyClass(
555                         proxyName, interfaces);
556                 try {
557                     proxyClass = defineClass0(loader, proxyName,
558                             proxyClassFile, 0, proxyClassFile.length);
559                 } catch (ClassFormatError e) {
560                     /*
561                      * A ClassFormatError here means that (barring bugs in the
562                      * proxy class generation code) there was some other
563                      * invalid aspect of the arguments supplied to the proxy
564                      * class creation (such as virtual machine limitations
565                      * exceeded).
566                      */
567                     throw new IllegalArgumentException(e.toString());
568                 }
569             }
570             // add to set of all generated proxy classes, for isProxyClass
571             proxyClasses.put(proxyClass, NULL_VALUE);
572 
573         } finally {
574             /*
575              * We must clean up the "pending generation" state of the proxy
576              * class cache entry somehow.  If a proxy class was successfully
577              * generated, store it in the cache (with a weak reference);
578              * otherwise, remove the reserved entry.  In all cases, notify
579              * all waiters on reserved entries in this cache.
580              */
581             if (proxyClass != null) {
582                 cache.put(key, new WeakReference<Class<?>>(proxyClass));
583             } else {
584                 cache.remove(key);
585             }
586 
587         }
588         return proxyClass;
589     }

591     /**
592      * Returns an instance of a proxy class for the specified interfaces
593      * that dispatches method invocations to the specified invocation
594      * handler.  This method is equivalent to:
595      * <pre>
596      *     Proxy.getProxyClass(loader, interfaces).
597      *         getConstructor(new Class[] { InvocationHandler.class }).
598      *         newInstance(new Object[] { handler });
599      * </pre>
600      *
601      * <p>{@code Proxy.newProxyInstance} throws
602      * {@code IllegalArgumentException} for the same reasons that
603      * {@code Proxy.getProxyClass} does.
604      *
605      * @param   loader the class loader to define the proxy class
606      * @param   interfaces the list of interfaces for the proxy class
607      *          to implement
608      * @param   h the invocation handler to dispatch method invocations to
609      * @return  a proxy instance with the specified invocation handler of a
610      *          proxy class that is defined by the specified class loader
611      *          and that implements the specified interfaces
612      * @throws  IllegalArgumentException if any of the restrictions on the
613      *          parameters that may be passed to {@code getProxyClass}
614      *          are violated
615      * @throws  NullPointerException if the {@code interfaces} array
616      *          argument or any of its elements are {@code null}, or
617      *          if the invocation handler, {@code h}, is
618      *          {@code null}
619      */
620     public static Object newProxyInstance(ClassLoader loader,
621                                           Class<?>[] interfaces,
622                                           InvocationHandler h)
623             throws IllegalArgumentException
624     {
625         if (h == null) {
626             throw new NullPointerException();
627         }
628 
629         /*
630          * Look up or generate the designated proxy class.
631          */
632         Class<?> cl = getProxyClass(loader, interfaces);
633 
634         /*
635          * Invoke its constructor with the designated invocation handler.
636          */
637         try {
638             Constructor cons = cl.getConstructor(constructorParams);
639             return cons.newInstance(new Object[] { h });
640         } catch (NoSuchMethodException e) {
641             throw new InternalError(e.toString());
642         } catch (IllegalAccessException e) {
643             throw new InternalError(e.toString());
644         } catch (InstantiationException e) {
645             throw new InternalError(e.toString());
646         } catch (InvocationTargetException e) {
647             throw new InternalError(e.toString());
648         }
649     }
